# 中文写作排版风格指南 - 完整规则

本文件包含完整的中文写作排版风格指南规则。这是 `/README.md` 的镜像，确保 skill 可以独立访问完整规则。

> **注意**：如需查看最新版本，请参阅项目根目录的 `README.md`。

## 中文

### 汉字规范

1. 使用简体字，推荐遵守《通用规范汉字表》。

### 用语

1. 使用中国大陆地区通行的词汇和惯用语，不使用港台地区词语。

   正确：算法、参数、数组、回调

   错误：演算法、引数、阵列、回呼

2. 不使用网络语言和网络流行语。

   错误：喜大普奔、不明觉厉、囧、焱暒妏

3. 不使用带歧视性的、不雅的语言。

4. 使用简练直白的语言。

5. 避免错别字。

   正确：登录、阈值、重启

   错误：登陆、阀值、重起

### 翻译

1. 使用中国大陆地区的译法，不使用港台地区的译法。

2. 当一个英文单词在没有广为接受的汉译词语或者使用其汉译词语会产生歧义时，可以使用英文单词本身。

   正确：在 debug 过程中，我们发现当参数为 `null` 时，会触发另外一个 bug。

3. 当一个英文缩写为人熟知并广为接受时，可以使用英文缩写而不使用其中文翻译。

   正确：RightCapital 的 CEO 拥有哥伦比亚大学的 MBA 学位。

4. 当要表达一个英文词汇，其中文翻译尚未被广为接受或存在歧义时，可以把英文原文写在括号里作为注释。在一篇文章中，同一个词汇只在其第一次出现的地方添加注释。

   正确：在常见的 HTTP 方法中，POST 和 PATCH 不是幂等（idempotent）的。

### 中英文混排

1. 在中文文章标题、段落和句子中出现的英文单词、短语和句子，其拼写、用法和风格遵守本指南中的英文部分。

2. 中文句子独立出现的英文名词，无论是表达一个还是多个，都使用该名词的单数形式。

   正确：这段代码中有好几个 bug。

   错误：这段代码中有好几个 bugs。

### 排版格式

1. 不使用段首缩进（例如段落左空二字）。

### 空格

1. 汉字与英文，汉字与阿拉伯数字之间应添加一个空格。

   正确：世界上有 10 种人，一种是懂二进制的，另一种是不懂的。

2. 汉字标点与英文，汉字标点与阿拉伯数字之间不应该添加空格。

   正确：在《The Hitchhiker's Guide to the Galaxy》中，生命、宇宙以及一切的终极答案是 42。

3. 汉字与半角标点之间不添加空格。

   正确：订阅的价格是 100 美元/月。

4. 汉字中格式化的部分（比如加粗、斜体、颜色、上下角标、超链接等）和未格式化的部分之间不应使用空格。

   正确：PHP 是世界上**最好**的语言。

### 标点符号和排版符号

1. 中文标点符号的用法，原则上遵守 GB/T 15834。

2. 中文句子中使用汉字标点符号。标点符号字符表：

   | 标点符号 | 字形 | Unicode | 注释 |
   |---------|-----|---------|------|
   | 句号 | 。 | U+3002 | 不使用 U+FF0E |
   | 逗号 | ， | U+FF0C | |
   | 顿号 | 、 | U+3001 | |
   | 感叹号 | ！ | U+FF01 | |
   | 问号 | ？ | U+FF1F | |
   | 冒号 | ： | U+FF1A | |
   | 分号 | ； | U+FF1B | |
   | 左引号 | " | U+201C | |
   | 右引号 | " | U+201D | |
   | 左书名号 | 《 | U+300A | |
   | 右书名号 | 》 | U+300B | |
   | 左括号 | （ | U+FF08 | |
   | 右括号 | ） | U+FF09 | |
   | 破折号 | —— | 两个 U+2014 | |
   | 省略号 | …… | 两个 U+2026 | |
   | 分隔号 | / | U+002F | 不使用 U+FF0F |

3. 中文段落中的句子中如果出现的英文语段，英文语段中标点符号应遵循英文规则。

   正确：RightCapital 的办公室文化是"Work smart, play hard, live well"。

4. 并列的词语之间如果是"或"的关系，可以使用分隔号"/"。

5. 并列词语之间应该使用顿号。

   正确：RightCapital 的技术部是由前端、后端、运维三个团队组成。

6. 并列词语之间，当最后两个词语之间使用"和"或者"或"这类连词，词语之间不应该仿照英文的牛津逗号使用顿号。

   正确：RightCapital 的技术部是由前端、后端和运维三个团队组成。

   错误：RightCapital 的技术部是由前端、后端、和运维三个团队组成。

7. 使用弯引号""""，不应使用直角引号"「」"。

8. 直角引号可以用于分割专有名词。

   正确：RightCapital 的内部 IT 系统是基于「零信任」原则构建。

9. 在中文句子中使用括号注释句中语段，如果括号中的内容和括号所修饰的内容都是英文，应使用半角括号，否则应使用全角括号。

   正确：RightCapital (a fintech company) 的手机应用（app）于近日发布。

10. 中文句子中的出现的出版物名称，无论中文还是英文，都应该使用书名号"《》"。

    正确：《三体》和《The Hitchhiker's Guide to the Galaxy》都是值得一读的科幻小说。

11. 汉字的区间和范围应使用一字线"—"或者波浪号"～"。

    正确：北京—上海的高铁最快仅需 4 小时 18 分。

12. 在分行列举中，如果并列各项不是完整句子时，除最后一项外，各项都应使用分号结尾；最后一条应使用句号结尾。

13. 在分行列举中，如果并列各项是完整句子时，每项都使用句号结尾。

14. 使用半角百分号。

    正确：在引入了新的优化之后，系统的性能得到了 15% 的提升。

15. 使用半角版本数学符号，包括负号"-"、加号"+"、减号"-"、乘号"*"、除号"/"等。

### 数字

1. 数字的用法原则上遵守 GB/T 15835。

2. 不超过 10 的数字推荐使用中文数字，10 和 10 以上的数字推荐使用阿拉伯数字。

3. 使用半角版本阿拉伯数字。

4. "万""亿"中文单位可以使用阿拉伯数字，其它中文单位只能使用汉字数字。

   正确：这个代码仓库有 300 万行代码。
   正确：这个代码仓库有三百行代码。

   错误：这个代码仓库有 3 百万行代码。

5. 中文句子中出现的阿拉伯数字时，如果数字是四位或者四位以上，应该使用半角逗号按千位分隔。

   正确：这个代码仓库有 3,000,000 行代码。

### 电话号码

1. 座机号码应该在局号和最后 4 位号码之间使用短横线或者空格进行分隔。

   正确：6123-4567 或 123 4567

2. 座机号码当包含区号时，区号需添加长途冠码 0，区号和号码之间使用一个空格。

   正确：010 6123-4567 或 (010) 6123-4567

3. 手机号应该按照 3-4-4 分组，使用短横线或空格分隔。

   正确：139-1234-5678

4. 400 或者 800 号码按照 3-3-4 分组。

   正确：400-123-4567

5. 座机号码当包含国际区号时，必须同时包含国内区号，且国际区号必须添加国际冠码，国内区号必须不添加长途冠码。

   正确：+86 10 6123-4567

### 日期和时间

1. 日期和时间的用法原则上遵守 GB/T 15835。

2. 日期应按照年月日顺序书写。

   正确：2020 年 3 月 31 日 或 2020-03-31

   错误：3 月 31 日 2020 年 或 2020/3/31

3. 年份应该用 4 位数。

4. 使用汉字"月""日"时，不编虚位（即 1 不编为 01）。

   正确：3 月 31 日

   错误：03 月 31 日

5. 时间中的时、分、秒使用半角冒号":"分隔。

   正确：上午 9:05

   错误：上午 9：05

## 英文

### 基本规则

1. 英文段落和中文段落中的英文单词、短语和句子，其语法、用法和风格原则上遵守《Chicago Manual of Style》。

### 拼写

1. 使用美式英语拼写。

   正确：Color, grey, center, canceled

   错误：Colour, gray, centre, cancelled

2. 商标和品牌名应该遵循其官方拼写。

### 大小写

1. 商标和品牌名应该遵循其官方拼写风格使用大小写，即使位于段首也不对其首字母大写。

   正确：iPhone 在短短数年内就挑战了 BlackBerry 在企业市场中的地位。

2. 文章的标题（title）、出版物名称应该使用标题大小写（title case）。

3. 章节的标题（heading）、表格和图表的标题、表格的表头应使用句子大小写。

4. 在条目列表中，当条目都是包含动词的完整句子时，所有条目的第一个单词的首字母都应该大写。

5. 避免使用全大写（all-caps）。

### 排版格式

1. 不使用段首缩进。

2. 出版物名称使用斜体。

### 空格

1. 数字和单位之间通常使用空格，百分号"%"、温度单位、角度单位除外。

   正确：5.0 cm, 45 kg, 32°C, 50%, 45°

2. 数字和倍数符号之间，倍数符号和单位之间都不使用空格。

   正确：128GB, 40Gb, 5GHz

### 标点符号和排印符号

1. 英文句子使用英文标点，不应使用汉字标点。

   | 标点符号 | 字形 | Unicode |
   |---------|-----|---------|
   | 撇号 | ' | U+2019 |
   | 左引号 | " | U+201C |
   | 右引号 | " | U+201D |
   | 省略号 | ... | 三个 U+002E |
   | 连字符 | - | U+002D |
   | En dash | – | U+2013 |
   | Em dash | — | U+2014 |

2. 英文句子中最后一个单词如果以点"."结尾，则该句不再使用句号。

   正确：The company's name is RightCapital Inc.

3. 括号的外侧需要保留一个空格。

4. 在三个或三个以上并列词组之间，在连词之前应该使用牛津逗号。

   正确：front-end, back-end, and DevOps teams

5. 破折号应使用 em dash "—"，左右不要留空格。

6. 表示区间时推荐使用 en dash "–"。

   正确：Fiscal Year 2018–2019

7. 在文章正文中不建议使用 ampersand "&"，应该展开为"and"，但专有名词不受此限。

### 货币

1. 货币前缀和数字之间不使用空格。

   正确：$12.34 或 US$1,234.56

2. 在财务/会计语境中，负数用括号表示，货币符号置于括号之外。

   正确：$(12.34)

### 数字

1. 四位和四位以上的数字应使用逗号作为千位分隔符。

2. 序数中的字母不应该以上角标出现。

   正确：1st, 2nd, 3rd

### 电话号码

1. 区号可以使用括号，当使用括号时区号和电话号码之间使用空格。

   正确：(212) 123-4567 或 212-123-4567

2. 国际电话号码应使用加号"+"作为国际冠码。

   正确：+1 (212) 123-4567

### 日期和时间

1. 使用美式风格的日期，按照月日年顺序书写。

   正确：January 31, 2021 或 1/31/2021

2. 日期中的日不使用序数。

   正确：January 31

   错误：January 31st

3. 12 小时制的时间，a.m./p.m. 使用小写，时间和 a.m./p.m. 之间使用一个空格。

   正确：9:30 a.m. 或 9:30 am

4. 午夜和正午 12 点应分别使用"12:00 midnight"和"12:00 noon"。

5. 12 小时制的时间，小时部分必须不编虚位；24 小时制必须使用虚位保持两位数字。

   正确：9:05 am, 09:05

## 代码

1. 变量类型、数据库字段类型、类、方法、函数、变量的名字、字面量在句子中应该格式化为代码。

   正确：这个函数会把 `$id` 参数保存到数据库中一个 `TINYINT UNSIGNED` 类型的字段中。

2. 字符串字面量当格式化为代码时，如果在句中不引起歧义，可以不加引号。

   正确：当 `$color` 参数为 `transparent` 的时候...
